# ລາຍງານສະຫຼຸບງານປະຈຳວັນ — Daily Production Dashboard

Dashboard แบบ static (HTML/CSS/JS ล้วน ไม่มี backend) ดึงข้อมูล real-time จาก Google Sheets 3 ไฟล์ พร้อม deploy ขึ้น GitHub Pages ได้ทันที

---

## 1. โครงสร้างไฟล์

```
lao-dashboard/
├── index.html          ← หน้าหลัก
├── css/style.css        ← ธีมแดง-ดำ, dark/light mode
├── js/config.js         ← ⭐ ตั้งค่าทั้งหมดอยู่ที่นี่ (API key, Sheet ID, ค่าคงที่)
├── js/sheets.js         ← โมดูลดึง+แปลงข้อมูลจาก Google Sheets API
├── js/app.js            ← render UI, กราฟ, refresh loop
└── README.md
```

## 2. วิธีสร้าง Google Sheets API Key (ทำครั้งเดียว ใช้เวลา ~5 นาที)

1. เข้า https://console.cloud.google.com/ แล้วสร้างโปรเจกต์ใหม่ (หรือใช้โปรเจกต์เดิม)
2. เมนูซ้าย → **APIs & Services → Library** → ค้นหา **"Google Sheets API"** → กด **Enable**
3. เมนูซ้าย → **APIs & Services → Credentials** → **Create Credentials → API key**
4. คัดลอกคีย์ที่ได้ (หน้าตาแบบ `AIzaSy...`)
5. **สำคัญมาก — จำกัดสิทธิ์คีย์เพื่อความปลอดภัย:**
   - กด "Edit API key"
   - หัวข้อ **API restrictions** → เลือก "Restrict key" → ติ๊กเฉพาะ **Google Sheets API**
   - หัวข้อ **Application restrictions** → เลือก "Websites" → ใส่โดเมน GitHub Pages ของคุณ เช่น `https://<username>.github.io/*`
   - บันทึก
   > ถ้าไม่จำกัดโดเมน ใครก็ตามที่เห็นคีย์ในซอร์สโค้ด (ซึ่งมองเห็นได้เสมอในเว็บฝั่ง client) จะเอาไปใช้ยิง Sheets ของคุณได้

6. นำคีย์ไปใส่ในไฟล์ `js/config.js` แทนที่ `YOUR_GOOGLE_SHEETS_API_KEY_HERE`

## 3. เปิดสิทธิ์ให้ Google Sheets ทั้ง 3 ไฟล์

ต้องทำทั้ง 3 ไฟล์ (ไฟล์ยอดผลิต / ไฟล์ออเดอร์ / ไฟล์คลังวัตถุดิบ):

`เปิดไฟล์ → กด Share (แชร์) → เปลี่ยนเป็น "Anyone with the link" → บทบาท "Viewer" → Done`

ถ้าไม่เปิดสาธารณะแบบนี้ Sheets API จะดึงข้อมูลไม่ได้แม้จะมี API key ถูกต้อง

## 4. จุดที่ "สมมติไว้" และควรตรวจสอบก่อนใช้งานจริง

ระบบนี้ดึงข้อมูลจาก **Named Range** สองตัว (`DAILY_ORDER`, `mond`) ที่โครงสร้างจริงในชีตอาจมีได้หลายแบบ โค้ดใน `js/sheets.js` ถูกเขียนให้ตรวจจับอัตโนมัติ 3 รูปแบบ:

| Named Range | รูปแบบที่รองรับอัตโนมัติ |
|---|---|
| `DAILY_ORDER` | (ก) เซลล์เดียว, (ข) หลายแถว 2 คอลัมน์ `[วันที่, ค่า]`, (ค) หลายแถว 1 คอลัมน์ (ถือว่าเป็น 7 วันล่าสุด จันทร์-อาทิตย์) |
| `mond` | (ก) 2 คอลัมน์ `[ชื่อเดือน, ค่า]`, (ข) 1 คอลัมน์ 12 แถว (ถือว่าเป็น ม.ค.-ธ.ค. ตามลำดับ) |

**วิธีตรวจสอบว่าระบบตีความถูกหรือไม่:** เปิดเว็บ → กด `F12` → แท็บ Console → ดู log ที่ขึ้นต้นด้วย `RAW:` จะเห็นข้อมูลดิบที่ API ส่งกลับมา แล้วเทียบกับตารางด้านบน

ถ้าโครงสร้างจริงไม่ตรง ให้แก้ฟังก์ชัน `parseDailyOrder()` หรือ `parseMonthlyStats()` ใน `js/sheets.js` ให้ตรงกับ Sheet จริงของคุณ (มีคอมเมนต์กำกับไว้ในโค้ดทุกจุด)

**ชื่อ Sheet tab รายเดือน:** ตั้งไว้เป็นชื่อเดือนภาษาลาวเต็ม (`ມັງກອນ`, `ກຸມພາ`, … `ກໍລະກົດ` สำหรับเดือนนี้) ที่ `CONFIG.LAO_MONTH_SHEET_NAMES` ใน `js/config.js` — ถ้าชื่อ tab จริงในชีตไม่ตรงตัวสะกดนี้เป๊ะๆ ให้แก้ array นี้ให้ตรงกับ tab จริง (คัดลอกชื่อ tab มาวางเลยจะชัวร์ที่สุด)

## 5. รันทดสอบก่อน deploy

เปิดไฟล์ `index.html` ด้วย Live Server (VS Code extension) หรือรัน:
```bash
cd lao-dashboard
python3 -m http.server 8000
```
แล้วเข้า `http://localhost:8000`

> ห้ามเปิดไฟล์ `index.html` แบบ double-click ตรงๆ (`file://`) เพราะ browser บาง engine จะบล็อก fetch — ต้องรันผ่าน local server หรือ deploy จริงเท่านั้น

## 6. Deploy ขึ้น GitHub Pages

```bash
git init
git add .
git commit -m "Daily production dashboard"
git branch -M main
git remote add origin https://github.com/<username>/<repo-name>.git
git push -u origin main
```

จากนั้น: GitHub repo → **Settings → Pages** → Source: **Deploy from a branch** → Branch: `main` / `(root)` → Save

รอ 1-2 นาที เว็บจะขึ้นที่ `https://<username>.github.io/<repo-name>/`

**อย่าลืม:** กลับไปที่ Google Cloud Console → แก้ Application restrictions ของ API key ให้เป็นโดเมนนี้จริง (ข้อ 2.5)

## 7. ปรับแต่งเพิ่มเติม

| ต้องการ | แก้ที่ไฟล์ |
|---|---|
| เปลี่ยนความถี่ real-time refresh | `CONFIG.REFRESH_INTERVAL_MS` ใน `js/config.js` |
| เปลี่ยนกำลังผลิต/เดือน (ใช้คำนวณ ETA) | `CONFIG.MONTHLY_PRODUCTION_CAPACITY` |
| เปลี่ยนราคาต่อชิ้น (ใช้คำนวณยอดขายประมาณการ) | `CONFIG.PRICE_PER_UNIT_KIP` |
| เปลี่ยนสีธีม | ตัวแปร CSS ใน `:root` และ `[data-theme="light"]` ในไฟล์ `css/style.css` |
| เปลี่ยนตำแหน่งเซลล์ L128 | `CONFIG.TODAY_PRODUCTION_CELL` |

## 8. อัปเดตล่าสุด — กราฟ 3 เส้น + ปุ่มบันทึกรายงาน

**กราฟ "ສະຖິຕິການຜະລິດລາຍອາທິດ"** เปลี่ยนจากแท่ง (bar) เป็นเส้นโค้ง 3 สี:
- 🔴 **ຜະລິດ** — ยอดผลิตสำเร็จรายวัน
- 🟢 **ອໍເດີໃໝ່** — ออเดอร์ใหม่รายวัน (ดึงจาก `DAILY_ORDER` ตรงๆ มีประวัติจริงในชีต)
- 🟠 **ອໍເດີຄ້າງ** — ออเดอร์ค้างรายวัน

⚠️ **ข้อจำกัดสำคัญ:** ข้อมูล "ຜະລິດ" และ "ອໍເດີຄ້າງ" ไม่มีประวัติรายวันเก็บไว้ในชีต (มีแค่ค่าล่าสุด) ระบบเลย**บันทึกค่าที่ดึงได้ในแต่ละวันไว้ใน `localStorage` ของเบราว์เซอร์เครื่องนั้นๆ** (ไฟล์ `js/history.js`) แล้วเอา 7 วันล่าสุดมาวาดกราฟ

ผลคือ:
- วันแรกที่เปิดใช้งาน กราฟจะมีจุดข้อมูลแค่ 1 จุด (วันนี้) แล้วค่อยๆ เพิ่มขึ้นทุกวันที่เปิดเว็บ จนครบ 7 วันภายในหนึ่งอาทิตย์
- ถ้าเปลี่ยนเครื่อง/เบราว์เซอร์/ล้าง cache ประวัติจะหายไป (เพราะเก็บไว้ในเครื่องนั้นเท่านั้น ไม่ได้ส่งกลับไปที่ Sheet)
- ถ้าต้องการประวัติที่แม่นยำและ sync ข้ามเครื่องได้จริง ควรให้ทีมงานบันทึกยอดผลิต/ยอดค้างรายวันเก็บไว้ในชีตเองทุกวัน (เพิ่มแถวใหม่ทุกวัน) แล้วทำ Named Range ใหม่ชี้ไปที่ช่วงนั้น — ถ้าต้องการแบบนี้แจ้งได้ จะช่วยปรับโค้ดให้ดึงจาก Sheet โดยตรงแทน

**ปุ่มใหม่ในหน้าเว็บ (footer):**
- **"ບັນທຶກຮູບໜ້າຈໍ"** — ถ่ายภาพหน้า dashboard ทั้งหมดเป็นไฟล์ `.png` ดาวน์โหลดทันที (ใช้ไลบรารี html2canvas) เหมาะกับส่งเข้า LINE/Telegram ให้หัวหน้า
- **"ພິມ / ບັນທຶກເປັນບິນ (PDF)"** — เปิดหน้าต่างพิมพ์ของเบราว์เซอร์ เลือก "Save as PDF" ได้ทันที (ตัดปุ่ม/เมนูที่ไม่จำเป็นออกอัตโนมัติตอนพิมพ์)

**ตัวเลขสำคัญเด่นชัดขึ้น:** ตัวเลข "ອໍເດີໃໝ່ວັນນີ້" และ "ອໍເດີຄ້າງທັງໝົດ" เปลี่ยนสีให้ตรงกับสีเส้นกราฟ (เขียว/ส้ม) พร้อม glow เบาๆ ให้อ่านง่ายขึ้น

## 9. เช็คลิสต์ตอนขึ้นปีใหม่ (ทำทุกสิ้นปี) ⭐

โครงสร้างจริงของทั้ง 2 ไฟล์ไม่เหมือนกัน ต้องจัดการคนละแบบ:

### 9.1 ไฟล์ Production (ตัด-ຫຍິບ, มี cell L128)
**ทุกปีจะเปลี่ยนเป็นไฟล์ใหม่ทั้งไฟล์ (Spreadsheet ID ใหม่)** ต้องแก้ `js/config.js` **2 จุด**:

1. เปิดไฟล์ Google Sheet ใหม่ของปีนั้น → คัดลอก ID จาก URL (ส่วนระหว่าง `/d/` กับ `/edit`)
2. แก้ใน `js/config.js`:
   ```js
   SHEETS: {
     PRODUCTION: "ใส่ ID ไฟล์ใหม่ตรงนี้",   // <-- จุดที่ 1
     ...
   },
   HEADER_LINKS: {
     ...
     CUT_SEW_SUMMARY: "https://docs.google.com/spreadsheets/d/ใส่ID ใหม่ตรงนี้/edit?gid=0#gid=0",  // <-- จุดที่ 2
   },
   ```
3. เช็คว่า tab เดือนในไฟล์ใหม่ยังตั้งชื่อเป็นภาษาลาวแบบเดิม (`ມັງກອນ`, `ກຸມພາ` ...) และ "ยอดผลิตวันนี้" ยังอยู่ที่ตำแหน่งเซลล์ `L128` เหมือนเดิม — ถ้าตำแหน่งเปลี่ยน ให้แก้ `TODAY_PRODUCTION_CELL` ด้วย
4. **เช็คว่าเปิดสิทธิ์ "Anyone with the link can view" ให้ไฟล์ใหม่แล้ว** (ไฟล์ใหม่ default จะเป็นส่วนตัว ต้องเปิดเองใหม่ทุกครั้ง)
5. Commit + push ขึ้น GitHub (อัปโหลดไฟล์ `config.js` แทนที่ไฟล์เดิม → Commit directly to `main`)

### 9.2 ไฟล์ Orders (DAILY_ORDER, TOTAL_ORDER, year, mond)
**ใช้ไฟล์เดิม ลิงก์เดิม แค่เพิ่ม tab ใหม่** (เช่น tab `2027`) — **ไม่ต้องแก้โค้ดเลย** แต่ต้องทำในตัว Google Sheet เอง:

1. สร้าง tab ปีใหม่ (แนะนำ: คลิกขวา tab ปีเก่า → Duplicate → เปลี่ยนชื่อเป็นปีใหม่ → ลบข้อมูลแถวเก่าออก เพื่อให้ตำแหน่งเซลล์เหมือนเดิมเป๊ะ)
2. เปิด **Data → Named ranges**
3. แก้ **Range** ของทั้ง 4 ตัว (`DAILY_ORDER`, `TOTAL_ORDER`, `year`, `mond`) ให้ชี้จาก `2026!...` เป็น `2027!...` (เปลี่ยนแค่ชื่อ tab หน้าเครื่องหมาย `!`)
4. กด Done — เว็บจะดึงข้อมูลจาก tab ใหม่ทันทีในการ refresh ครั้งถัดไป (ไม่ต้องแตะโค้ดเว็บเลย)

### 9.3 ไฟล์ Warehouse (ปุ่ม header เท่านั้น ไม่ได้ดึงข้อมูล)
ถ้าไฟล์นี้เปลี่ยนเป็นไฟล์ใหม่ทุกปีด้วยเหมือนกัน ให้แก้ `HEADER_LINKS.WAREHOUSE` และ `SHEETS.WAREHOUSE` ใน `config.js` ตามไฟล์ใหม่ (ถ้าใช้ไฟล์เดิมตลอด ไม่ต้องทำอะไร)
